## Введение

Данный документ описывает план тестирования приложения "Anime Collection Manager". Документ предназначен для людей, выполняющих тестирование данного проекта. Цель тестирования - проверка соответствия реального поведения программы проекта и ее ожидаемого поведения, которое описано в [требованиях](https://github.com/snrteftelya/verserkkk_app/blob/main/Requirements/SRS.md).

<a name="items"/>

## Объект тестирования

В качестве объектов тестирования можно выделить следующие функциональные требования:

* Управление пользователями (создание, редактирование, удаление, поиск);
* Управление каталогом аниме (создание, редактирование, удаление, поиск);
* Управление коллекциями (создание, редактирование, удаление, управление составом аниме);
* Навигация между связанными сущностями.

<a name="quality"/>

## Атрибуты качества

1. Функциональность:
   - функциональная полнота: приложение должно выполнять все заявленные функции;
   - функциональная корректность: приложение должно выполнять все заявленные функции корректно;
2. Удобство использования:
   - интуитивно понятный интерфейс на основе Material-UI;
   - быстрый отклик на действия пользователя;
   - корректная валидация вводимых данных.
3. Производительность:
   - время загрузки страниц менее 1 секунды;
   - время ответа API менее 300 мс.

<a name="risk"/>

## Риски

В данном случае к рискам можно отнести:
* Некорректная работа при отсутствии подключения к backend API;
* Проблемы с производительностью при большом объеме данных (>10000 записей);
* Ошибки валидации данных на границе frontend-backend;
* Проблемы совместимости с различными браузерами.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения. Аспекты, подвергаемые тестированию:

### Управление пользователями
* Создание нового пользователя
* Редактирование существующего пользователя
* Удаление пользователя
* Поиск пользователей по username/email
* Валидация обязательных полей

### Управление аниме
* Создание новой записи аниме
* Редактирование существующего аниме
* Удаление аниме
* Поиск аниме по названию, жанру, году
* Валидация данных (год выпуска, обязательные поля)

### Управление коллекциями
* Создание коллекции для пользователя
* Редактирование коллекции (название, состав аниме)
* Удаление коллекции
* Добавление аниме в коллекцию
* Удаление аниме из коллекции
* Просмотр коллекций пользователя

### Навигация и интеграция
* Переход от пользователя к его коллекциям
* Связи между пользователями, коллекциями и аниме
* Обработка ошибок API
* Responsive-дизайн на различных устройствах

### Создание пользователя
Этот вариант использования необходимо протестировать на:
* успешное создание пользователя с корректными данными;
* валидацию обязательных полей (username, email);
* валидацию формата email;
* проверку уникальности username и email;
* отображение сообщений об ошибках.

### Редактирование пользователя
Этот вариант использования необходимо протестировать на:
* загрузку текущих данных пользователя;
* успешное обновление данных;
* валидацию изменяемых данных;
* отмену редактирования без сохранения.

### Удаление пользователя
Этот вариант использования необходимо протестировать на:
* подтверждение удаления через диалоговое окно;
* каскадное удаление связанных коллекций;
* обновление списка после удаления.

### Создание аниме
Этот вариант использования необходимо протестировать на:
* успешное создание аниме с корректными данными;
* валидацию обязательного поля title;
* валидацию года выпуска (1900-текущий год);
* обработку некорректных данных.

### Поиск аниме
Этот вариант использования необходимо протестировать на:
* поиск по полному названию;
* поиск по части названия;
* поиск по жанру;
* поиск по году выпуска;
* комбинированный поиск.

### Создание коллекции
Этот вариант использования необходимо протестировать на:
* создание коллекции для выбранного пользователя;
* добавление аниме в коллекцию при создании;
* валидацию названия коллекции;
* проверку уникальности имени коллекции для пользователя.

### Управление составом коллекции
Этот вариант использования необходимо протестировать на:
* добавление аниме в существующую коллекцию;
* удаление аниме из коллекции;
* отображение текущего состава коллекции.

## Нефункциональные требования:
* Производительность: время загрузки страниц < 1 сек, время ответа API < 300 мс
* Совместимость: работа в Chrome, Firefox, Safari, Edge (последние 2 версии)
* Responsive-дизайн: корректное отображение на экранах 320px-4K
* Удобство использования: интуитивный интерфейс на основе Material-UI

<a name="approach"/>

## Подходы к тестированию

При тестировании будет использован комбинированный подход:
* **Ручное тестирование** - для проверки UI/UX и основных пользовательских сценариев
* **Автоматизированное тестирование** - для регрессионного тестирования API endpoints
* **Нагрузочное тестирование** - для проверки производительности при росте данных
